GitHub+Hexo搭建个人博客

Posted on 2018-08-28 | In Blog搭建 | Visitors:
Words count in article: 2,041 | Reading time ≈ 8

        很早就想弄一个博客了，觉得有一个属于自己的网站是件很酷很炫很装X的事。所以前段时间在阿里上注册了一个域名买了一个服务器，也大概搭了一个简单的页面，但是感觉要完全搭出一个很酷的页面还是挺费时间的。于是在知乎上搜了下，给我推荐了这篇文章。
        这里我只是记录了建站的一些主要步骤，如果你要问我Github、Hexo等是什么，可以参考前面的链接。好了，废话不多说进入正题吧！

搭建步骤：

  • Github创建仓库
  • 安装Git
  • 安装Node.js
  • 安装Hexo
  • 推送网站
  • 绑定域名
  • 更换主题
  • 使用MarkDown写博客
Github创建仓库

在Github官网注册账号，在Repositories里面新建一个仓库，仓库命名方式为：用户名.github.io，用户名为你的Github账号。

安装Git

在Git官网下载Git安装。
在搜索栏搜索Git Bash或者在桌面上右击鼠标找到Git Bash Here，输入git测试是否安装成功，若安装失败卸载重装或者根据原因自行百度。
安装成功后将Git和Github账号绑定，输入下面命令：
        git config --global user.name "你的GitHub用户名"
        git config --global user.email "你的GitHub注册邮箱"
生成SSH密钥：
        ssh-keygen -t rsa -C "你的GitHub注册邮箱"
在你的电脑中找到.ssh这个文件夹（看上面命令给你分配的路径），将其中id_rsa.pub文件用记事本打开，复制内容，打开GitHub的Settings页面，选择New SHH key，将刚复制好的文件放到key一栏中，title随便填写，最后点击Add SSH key。
在Git Bash中检验Github公钥是否设置成功，输入命令：ssh git@github.com,现实successfully就说明成功设置了。

安装Node.JS

Hexo基于Node.js，这部分都是在搭建Hexo需要的环境，Node.js下载地址,选择安装Node.js包包含环境变量和npm的安装。检验Node.js是否安装成功，输入node -v会显示当前版本。检验npm是否安装成功，输入npm -v会显示当前版本。

安装Hexo

Hexo是我们个人博客的框架。在电脑中新建一个文件夹，以后我们的博客都可以放在这个文件夹里面。然后在这个文件夹中，按住shift键并右击鼠标，打开Git Bash，输入npm命令npm install -g hexo-cli安装Hexo（时间有点长）。安装完成后我们就可以开始建立博客了。
首先，初始化博客。使用命令hexo init blog，这条命令会初始化一个博客，并在刚才那个文件夹里创建一个名称为blog的文件夹。为了检测网站雏形，可以使用下列命令，

hexo new test_my_site
hexo g
hexo s

上述命令作用分别为：创建一个名称为test_my_site的博客，生成博客，启动服务。

一个小坑
不过使用这些命令之前**先输入命令`cd blog`**。
这里很重要，我在这个地方栽了一个很大坑，
如果没有进入blog文件夹，后面命令会出现`Usage: hexo <command>`接一堆help文档的信息，
所以一定要进入blog文件夹在进行后面操作。

以上操作成功的话，你再网址栏输入localhost:4000就可以看到我们的博客网页了，是不是很激动，哈哈。
常用Hexo命令：

npm install hexo -g #安装Hexo
npm update hexo -g #升级 
hexo init #初始化博客

命令简写
hexo n "我的博客" == hexo new "我的博客" #新建文章
hexo g == hexo generate #生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy #部署

hexo server #Hexo会监视文件变动并自动更新，无须重启服务器
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP
hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令
推送网站

推送网站，也就是发布网站，能够在浏览器里输入url访问啦。下面我们将Hexo和Github关联起来（让Hexo知道你要把blog部署在哪个位置），打开blog文件夹里的_config.yml文件（站点配置文件），将文件末尾修改为：

deploy: 
type: git
repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git
branch: master参考如下：

接下来我们要安装Git部署插件，输入命令：npm install hexo-deployer-git --save。

一个小坑
安装deployer-git出现包不在维护This package is no longer maintained的问题
原因：blog下node_modules文件夹下发现“hexo-deployer-git”文件异常
解决：使用淘宝npm镜像
而使用淘宝npm时由于要使用命令cnpm install hexo-deployer-git --save
又会出现command not found错误
问题在于 你没用用淘宝的镜像 安装cnpm
解决办法npm install cnpm -g --registry=https://registry.npm.taobao.org

这个坑我也栽了好久，翻了好多博客才找到解决办法。
安装好Git插件后，我们就可以部署我们的网站了，输入命令：

hexo clean 
hexo g 
hexo d

最后，我们在浏览器中输入你放置个人网站的仓库路径 xxxx.github.io ，然后就可以查看到页面信息了。此时，恭喜你的博客上线了。

绑定域名

如果你想使用个性化的域名来访问你的博客，这里就需要进行绑定域名操作了。我使用的是阿里云.top的域名，价格相对还是很便宜的（5块钱），土豪可以考虑.com这些。购买域名后可能会需要实名注册，如果有登记一下信息就行了。
进入阿里云的控制台-域名，选择你的域名，进入解析，添加下面三条：

192.30.252.153          选择A记录
192.30.252.154          选择A记录
你的用户名.github.io     选择CNAME记录

然后进入Github之前建立的仓库，进入settings项，设置里面的Custom domain，输入你购买的域名，保存。
最后在本地博客文件夹blog/source目录下，创建一个记事本文件，输入你的域名，文件保存为所有格式，命名为CNAME。
文件中如果带有www，那么以后访问的时候必须带有www完整的域名才可以访问，但如果不带有www，以后访问的时候带不带www都可以访问。所以建议，不要带有www。
设置好后，我们就可以进入blog文件夹进行部署，在blog文件夹，打开Git Bash，输入命令：

hexo clean
hexo g
hexo d

最后在浏览器输入你的域名进行访问，哇咔，你自己的博客耶。
如果最后不能正常访问网站，出现404错误，检查前面过程是不是操作有误。我当时在Custom domain里面输错了域名（域名和我github昵称差了一个字母），找了好久原因，所以一定要细心。

更换主题

要修改Hexo的默认主题，在Hexo主题里面选择,我这里选择的是next主题
所以输入命令git clone https://github.com/iissnan/hexo-theme-next themes/next，添加next主题进去。然后通过theme: next选择该主题。
如果要选择next主题的样式，可以打开blog\themes\next文件夹中的主题配置文件_config.yml，找到scheme，修改样式（去掉#就是选中该样式）。
选择好后，输入命令hexo g和hexo d重新部署一下就可以。
